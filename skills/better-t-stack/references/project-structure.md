# BTS Project Structure

## Standard Layout

```
apps/
  web/              Next.js app (App Router, Route Handlers as API)
  native/           Expo / React Native app (Expo Router)
  docs/             Documentation site (Fumadocs or Starlight) — optional
  cli/              Internal CLI scripts (ingestion, release tooling) — optional

packages/
  api/              tRPC or ORPC routers, service classes, request context
  auth/             Better Auth config, session types, auth client exports
  db/               Prisma or Drizzle schema, migrations, db client
  env/              Validated environment contracts (split by runtime)
  theme/            Shared design tokens, TailwindCSS preset, fonts
  emails/           React Email templates, email sending utility
  ui/               Shared UI components — if web and native share components
  config/           Base tsconfig, biome config — if not using tooling/ tier

tooling/            (optional — collapse into packages/config for simpler projects)
  typescript/       Base tsconfig.json presets
  tailwind/         Shared TailwindCSS config
  biome/            Shared biome.json
```

---

## Key Files at Root

| File | Purpose |
|---|---|
| `package.json` | Workspace definition, root scripts, catalog |
| `turbo.json` | Turborepo pipeline config |
| `bts.jsonc` | BTS scaffold config (do not edit manually) |
| `biome.json` | Shared linting/formatting config |
| `.lefthook.yml` | Git hook definitions |
| `CLAUDE.md` / `AGENTS.md` | Agent instructions (generated by Ruler) |
| `.ruler/` | Ruler source templates for agent instruction files |
| `.mcp.json` | MCP server config |

## `packages/api` Structure

```
packages/api/
  src/
    index.ts          tRPC/ORPC router root export
    context.ts        Request context factory (see api-service-layer skill)
    router/
      _app.ts         Root router merging all sub-routers
      user.ts         User-domain router
      auth.ts         Auth-domain router (thin — delegates to packages/auth)
    services/
      user.service.ts  UserService class
      email.service.ts EmailService class
    providers/
      email/
        email.provider.ts   Abstract email provider interface
        resend.adapter.ts   Resend implementation
        mailpit.adapter.ts  Mailpit (dev) implementation
```

## `packages/db` Structure (Prisma)

```
packages/db/
  prisma/
    schema/
      schema.prisma      Main schema (or split into multiple files)
    migrations/          Generated migration files
  src/
    index.ts             Re-exports client and schema types
    client.ts            Prisma client singleton
  generated/             Prisma client output (gitignored)
```

## `packages/db` Structure (Drizzle)

```
packages/db/
  src/
    index.ts             Re-exports
    client.ts            Drizzle client
    schema/
      users.ts
      sessions.ts
  drizzle/               Migration SQL files
  drizzle.config.ts      Drizzle Kit config
```

## `packages/env` Structure

```
packages/env/
  src/
    server.ts     Server-only env (DATABASE_URL, secrets, etc.)
    web.ts        Next.js env (NEXT_PUBLIC_* + server imports)
    native.ts     Expo env (EXPO_PUBLIC_* only)
    index.ts      Re-exports all three
```

Each file uses `@t3-oss/env-nextjs` or `@t3-oss/env-core` with Zod validation. The split prevents server secrets from being accidentally bundled into client or native builds.

## `packages/auth` Structure

```
packages/auth/
  src/
    index.ts            Re-exports (server auth instance, session types)
    lib/
      auth.ts           Better Auth configuration (plugins, providers, session)
    auth-client.ts      Better Auth client (for web/native)
    types.ts            Session type exports
```
