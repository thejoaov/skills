# BTS Tooling Setup: Biome, Lefthook, Ruler

## Biome

BTS projects use Biome for linting and formatting instead of ESLint + Prettier. Single config, faster execution.

### Root `biome.json`

```json
{
  "$schema": "https://biomejs.dev/schemas/2.0.0/schema.json",
  "vcs": {
    "enabled": true,
    "clientKind": "git",
    "useIgnoreFile": true
  },
  "files": {
    "ignoreUnknown": true
  },
  "formatter": {
    "enabled": true,
    "indentStyle": "space",
    "indentWidth": 2,
    "lineWidth": 100
  },
  "linter": {
    "enabled": true,
    "rules": {
      "recommended": true,
      "correctness": {
        "noUnusedImports": "error",
        "noUnusedVariables": "error"
      }
    }
  },
  "javascript": {
    "formatter": {
      "quoteStyle": "double",
      "trailingCommas": "all",
      "semicolons": "asNeeded"
    }
  },
  "overrides": [
    {
      "include": ["*.ts", "*.tsx"],
      "linter": {
        "rules": {
          "suspicious": {
            "noExplicitAny": "warn"
          }
        }
      }
    }
  ]
}
```

### Running Biome

```bash
bun run lint          # Check
bun run lint:fix      # Fix auto-fixable issues
bun run format        # Format all files
bun run format:check  # Check format without writing
```

Via Turborepo, `lint` depends on `^build` to ensure type info is available.

## Lefthook

Lefthook runs Git hooks. BTS uses it to enforce quality gates before commits and pushes.

### `.lefthook.yml`

```yaml
pre-commit:
  parallel: true
  commands:
    lint:
      run: bun run lint
    type-check:
      run: bun run type-check
    format:
      run: bun run format:check

commit-msg:
  commands:
    commitlint:
      run: bunx commitlint --edit {1}
```

Install hooks after cloning:
```bash
bunx lefthook install
```

## Ruler

Ruler syncs agent instruction files (CLAUDE.md, AGENTS.md, WARP.md, cursor rules, windsurf rules, etc.) from a single source of truth.

### `.ruler/ruler.toml`

```toml
default_agents = ["copilot", "claude", "codex", "cursor", "windsurf", "gemini-cli", "warp"]

[agents.agentsmd]
path = "AGENTS.md"

[agents.claude]
path = "CLAUDE.md"

[agents.cursor]
path = ".cursor/rules/project.mdc"

[agents.windsurf]
path = ".windsurf/rules/project.md"
```

### Source file

The source is typically `.ruler/bts.md` (for BTS projects) or a custom `.ruler/instructions.md`. Edit only the source — Ruler writes all the target files.

```bash
bunx ruler generate   # Sync all agent files from the source
```

**Rule:** Never manually edit `CLAUDE.md`, `AGENTS.md`, or agent-specific rule files — they are generated by Ruler. Edit `.ruler/<source>.md` instead.
